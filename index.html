var

1. var can be redeclared and reinitailize as many times you want.
2. variable can be hoist which meaning the declearation move at the top of the scope

let

1. let can't be redeclared but can be reinitailize as many times you want.
2. let variable cannot be hoisted variable goes into temporal dead zone if we try to access the variable before decration.
3. let has a block scope
4. let is introdunce in ES6

const

1. const can't be redeclared or reinitailiz.
2. const variable cannot be hoisted variable goes into temporal dead zone if we try to access the variable before decration.
3. const has a block scope
4. const is introdunce in ES6
